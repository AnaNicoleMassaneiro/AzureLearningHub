Resumo: Serviço de Aplicativo do Azure

Explorar o Serviço de Aplicativo do Azure

O que é o Serviço de Aplicativo?

  • Serviço gerenciado para hospedar aplicativos web, APIs, funções e backends móveis.
  • Suporte para várias linguagens: C#, Java, Python, Node.js, PHP e mais.
  • Recursos principais:
    • Escalabilidade automática.
    • Hospedagem globalmente distribuída.
    • Suporte integrado para autenticação e autorização.

Planos do Serviço de Aplicativo

  • Camadas disponíveis:
    • Gratuita/Compartilhada: Para teste e desenvolvimento.
    • Básica: Aplicativos de produção pequenos.
    • Standard: Mais capacidade de escalabilidade.
    • Premium: Performance avançada.
    • Isolada: Ambiente dedicado em rede virtual.
  • Escolha o plano com base em requisitos de desempenho, escalabilidade e conectividade.

Implantação no Serviço de Aplicativo

  • Métodos:
    • Azure CLI: az webapp up para subir um app rapidamente.
    • GitHub Actions ou Azure DevOps: Automação de CI/CD.
    • FTP e Zip Deploy: Métodos manuais.
  • Comandos úteis:
    • Criar um app: az webapp create
    • Configurar plano: az appservice plan create

Autenticação e Autorização

  • Autenticação integrada: Suporte para provedores como Azure AD, Google, Facebook e Twitter.
  • Configurável sem alterações no código.
  • Autorização de nível de aplicação:
    • Restrições por IP.
    • Regras baseadas em tráfego.

Recursos de Rede

  • Rede Virtual: Acesso a recursos internos.
  • Hybrid Connections: Conectar-se a serviços locais.

Definir Configurações do Aplicativo Web

Configurações do Aplicativo

  • Variáveis de ambiente para gerenciar configurações de produção/teste.
  • Configurar com az webapp config appsettings set.

Configurações Gerais

  • Idioma e pilha de execução: .NET, Java, PHP etc.
  • Configuração de HTTPS obrigatório.

Mapeamento de Caminhos

  • Aplicativos virtuais: Dividir diretórios para diferentes aplicativos.

Log de Diagnóstico

  • Logs de eventos, solicitações HTTP e diagnósticos de falhas.
  • Habilitar logs com az webapp log config.

Certificados de Segurança

  • SSL/TLS para criptografia.
  • Importar certificados com suporte para HTTPS.

Escalar Aplicativos no Serviço de Aplicativo

Dimensionamento Horizontal

  • Adiciona mais instâncias do aplicativo.
  • Configurado com escalabilidade automática:
    • Baseado em CPU, memória, tráfego HTTP.

Dimensionamento Vertical

  • Atualiza o plano para mais recursos (ex.: mudar de Básico para Standard).

Práticas Recomendadas

  • Use o escalonamento automático para economizar custos.
  • Monitore o desempenho com Azure Monitor.

Explorar os Slots de Implantação

O que são Slots de Implantação?

  • Ambientes separados dentro de um mesmo Serviço de Aplicativo.
  • Tipos:
    • Produção: Slot principal.
    • Preparo (Staging): Testes antes de enviar para produção.

Troca de Slots

  • Realizar a troca de ambientes sem causar downtime.
  • Configuração: Configurações específicas podem ser vinculadas a slots ou globais.

Roteamento de Tráfego

  • Dividir o tráfego entre slots para validação gradual.
